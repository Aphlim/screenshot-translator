## 1. 项目背景(一句话)

FuckEnglish 是一个 Windows 桌面工具:用户按 `Ctrl+Alt+T` → 框选屏幕上的英文 → 应用通过 PaddleOCR 识别 + LLM 翻译 → 在框选位置旁边弹出译文窗口。基于 Electron + TypeScript + React,常驻系统托盘运行。

当前要做的是**设置页**:用户在这里配置 LLM API 信息(`baseURL` / `apiKey` / `model`),不再用文本编辑器手改 JSON。

## 3. 设置页功能与字段(必须覆盖)

整个页面只有**一个"翻译 API"卡片**(MVP 阶段),包含以下结构:

### 3.1 顶栏(同译文弹窗风格)
- 左:`●` 品牌渐变点 + `FuckEnglish` + `· 设置`
- 右:`关闭` 按钮(点击关闭窗口)

### 3.2 区段:翻译 API 配置

按顺序排列以下三个字段(每个字段有 label、input、可选 helper):

**字段 1:接口地址(Base URL)**
- Label: `接口地址`
- Helper:`OpenAI 兼容服务的根 URL`
- Input(text):占位符 `https://api.deepseek.com`
- 字段下方放 4 个**预设标签 chip**(点击填入对应 URL):
  - `DeepSeek` → `https://api.deepseek.com`
  - `OpenAI` → `https://api.openai.com/v1`
  - `SiliconFlow` → `https://api.siliconflow.cn/v1`
  - `Moonshot` → `https://api.moonshot.cn/v1`

**字段 2:API Key**
- Label: `API Key`
- Helper:`仅保存在本机,不会上传`
- Input(password):占位符 `sk-...`
- Input 右侧有个**眼睛 icon 切换显示/隐藏**(可以用 emoji `👁` 或简单的 SVG)

**字段 3:模型(Model)**
- Label: `模型`
- Helper:`要使用的模型 ID`
- Input(text):占位符 `deepseek-chat`

### 3.3 区段:操作

- 主按钮:`测试连接`(点击发起一个测试翻译请求,根据结果显示成功/失败提示)
- 次按钮:`保存并关闭`
- 第三按钮(更低优先级,文字按钮样式):`打开配置文件夹` —— 用浏览器风格的图标 + 文字小链接呈现

### 3.4 状态反馈区(顶栏下方,横贯一行)

测试连接 / 保存 之后这里出现一条提示,样式像浏览器 banner:
- 成功(绿):`✓ 连接成功,响应耗时 832ms`
- 失败(红):`✗ 401 Unauthorized — 请检查 API Key`
- 信息(蓝灰):`✓ 已保存`

3 秒后自动淡出(动画请用 opacity transition,不要 collapse 占位)。

### 3.5 首次启动引导(可选,如果有余力)

如果是用户首次打开应用、还没有保存任何 API key,设置页打开时顶部多一行欢迎提示:
> 欢迎使用 FuckEnglish 👋 在下方填入你的 API 配置即可开始。推荐用 DeepSeek(便宜,中国大陆访问快)。

样式比常规 banner 更柔和(浅紫底,带渐变,圆角 8px)。

## 4. 窗口与布局约束

- 窗口尺寸:**480 × 580 像素**(固定,不可拉伸)
- 不是全屏,不是透明:**普通带边框窗口**,用户能从任务栏看到、能用系统按钮关闭
- 单列布局,垂直滚动(如果内容超出)
- 整个内容区域有 14px 的外边距(让卡片有阴影呼吸感),内部主卡片继承之前的 12px 圆角 + 阴影
- 不要左侧导航栏,不要 tab,不要分屏

## 6. 给 Gemini 的具体交付要求

请产出**一份 HTML 文件**,包含:
- 完整的 `<html>` / `<head>`(把所有 CSS 写在 `<style>` 标签里,不要外部依赖)
- `<body>` 里直接渲染设置页 UI
- 至少展示 3 种状态:
  1. 初次打开(欢迎提示 + 空表单)
  2. 测试连接成功(绿色 banner)
  3. 测试连接失败(红色 banner)

可以用 `<div>` 模拟多个并排状态(类似 Storybook 那种 catalog 风格),方便我看不同状态。

**不要做的事**:
- ❌ 不要用 Tailwind / Bootstrap / 任何外部框架
- ❌ 不要写 JS 交互(我会自己接)
- ❌ 不要用 emoji 替代品牌区分(只有一个"品牌点"位置可以保留紫蓝渐变)
- ❌ 不要引入新色板 / 字体
- ❌ 不要做 dark mode(本应用目前只有浅色)

**可以发挥的点**:
- 输入框的微观细节(focus 时的边框 / outline、placeholder 样式)
- 按钮的不同状态(默认 / hover / active / disabled / loading)
- 测试连接时的 loading 动画(可以用脉动小点,延续译文弹窗的 `.dot` 动画)
- chip(预设标签)的样式
- 整体留白与对齐节奏
